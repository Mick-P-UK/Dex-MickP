## File Naming Convention - MANDATORY (ALL projects, every vault)
- Every dated file I create MUST follow this pattern:
  YYYY.MM.DD - Title - [event date]_vX.Y.ext
- FRONT: the date the file was MADE (today's date), YYYY.MM.DD, dots as separators.
  Always the creation date, not the event date.
- MIDDLE: a clear, descriptive title (keep " - " separators).
- END: the version as _vX.Y (e.g. _v2.0). If the file relates to an event on a
  specific date (e.g. a webinar), put that EVENT date immediately before the
  version, joined with an underscore: [event date]_vX.Y.
- Worked example - a script made 25 Jun 2026 for the 1 Jul 2026 webinar, v2.0:
  2026.06.25 - SCRIPT - Own It (Main Six-Clip Section) - 2026.07.01_v2.0.docx
- Version rules: start at _v1.0; bump minor (_v1.1) for small edits, major (_v2.0)
  for substantial rewrites. When superseding, move the old file to an _archive/
  _superseded/ folder keeping its full versioned name - NEVER silently overwrite.
- Applies to docx, md, pdf, xlsx, pptx and any dated deliverable.
- Exceptions: anchor files that must keep fixed names (CLAUDE.md, PROMPT_LIBRARY.md)
  and Notion pages (own YYYY.MM.DD - Title naming).
