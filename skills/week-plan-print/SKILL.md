# Week Plan Print Skill

Generates a clean, print-ready A4 Word document of the current week's calendar events for Mick to pin on the wall.

## Trigger

- `/week-plan-print` command
- "print the week plan"
- "weekly planner document"
- "wall planner"
- "printable schedule"

---

## Workflow

### Step 1: Verify Dates

ALWAYS run date verification first. Never calculate dates mentally.

```python
import calendar
from datetime import datetime, timedelta

today = datetime.now()
print(calendar.month(today.year, today.month))

monday = today - timedelta(days=today.weekday())
sunday = monday + timedelta(days=6)
print(f"Today: {today.strftime('%A %d %B %Y')}")
print(f"Week: {monday.strftime('%Y-%m-%d')} to {sunday.strftime('%Y-%m-%d')}")
```

### Step 2: Fetch Calendar Events

Fetch from both primary and Todoist calendars for the full 8-day window (today through end of next Sunday):

```
calendarId: ["mickp.dbox@gmail.com", "mjgmfqaaltk8b519uunvr1hjug@group.calendar.google.com"]
timeMin: today at 00:00:00
timeMax: next Sunday at 23:59:59
timeZone: Europe/London
```

### Step 3: Organise Events by Day

Group events by date. For each event extract:
- Start time (format as 12hr: "9:00am", "7:30pm")
- Summary/title
- Any notes (location, flags like "due shortly!")

Skip: all-day birthday/anniversary events from contacts (show only if it is someone Mick knows personally). Skip any untitled/blank events.

### Step 4: Generate the DOCX

Use the docx npm library. See `/mnt/skills/public/docx/SKILL.md` for full reference.

Key layout rules:
- Paper: A4 (11906 x 16838 DXA), margins 0.5 inch (720 DXA) all sides
- Title bar: "WEEKLY PLANNER  |  [Date range]" centred, dark blue, with bottom border
- One table spanning full width (8960 DXA), two columns: time (1800 DXA) + event (7160 DXA)
- Day headers: Full-width merged row, dark blue background (#1F3864), white bold text
- Highlight the current day header in mid-blue (#2E5FA3)
- Highlight any WEBINAR or major event days with a note in the header
- Alternating row shading: white / light grey (#F2F2F2)
- Time column: bold mid-blue (#2E5FA3), 18pt Arial
- Event column: dark text (#1A1A1A), 18pt Arial; notes in smaller italic grey
- Spacer rows between each day
- Flag important events with a short note (e.g. insurance renewals, medical appointments)
- Footer: right-aligned, small italic grey: "Generated by Cedric  |  DIY Investors PAIDA  |  [date]"

See `references/colour-palette.md` for full colour reference.
See `scripts/generate_week_doc.js` for the base JS template (copy, adapt for current week data, run with node).

### Step 5: Output

1. Save to `/home/claude/week_plan_[date_range].docx`
2. Copy to `/mnt/user-data/outputs/`
3. Use `present_files` tool to deliver to Mick
4. Confirm delivery with a one-line summary

## Important Flags to Highlight

When scanning events, automatically add a short note for:
- Insurance renewals, financial deadlines
- Medical/dental/chiro appointments
- "Due shortly!" or similar urgency markers in event titles
- Events marked with reminder overrides (often indicate importance)
- First occurrence of webinar prep days

---

## Error Handling

- If calendar fetch fails: report the error, do not generate with assumed data
- If a day has no events: include the day header with "No events scheduled"
- If docx generation fails: check npm docx is installed (npm list -g docx), install if missing
