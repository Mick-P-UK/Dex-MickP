## 2. Why this architecture (mirror-and-push personal repo)

The problem: your personal folders (00-Inbox, 04-Projects, 05-Areas, etc.) live
INSIDE the Dex-MickP folder, which is already a git repo. Two git repos cannot
cleanly track the same folders at the same level. Overlaid-git tricks exist but are
fragile and would fight the existing ASCII pre-commit hook and the recurring
index.lock problem.

The chosen solution: a SEPARATE private repo in its OWN folder, fed once a night by
a copy-then-commit-push of your personal folders out of the live vault.

- Live content stays exactly where it is in Dex-MickP (nothing moves out of your
  working vault).
- The framework fork stays pristine and shareable.
- The personal repo has NO ASCII hook, so notes with any characters back up freely.
- It reuses the same 9pm Windows Task Scheduler pattern you already trust.
- Trade-off: the personal content is duplicated on disk (small - it is text) and the
  backup is a nightly snapshot, not live.

## 5. Phase B - Personal backup repo (runs on Mick's PC)

All commands are PowerShell. Windows paths that contain an apostrophe are
double-quoted. Run each block WHERE indicated.

### B1. Create the private GitHub repo (WHERE: github.com in your browser)

1. Go to https://github.com/new
2. Owner: Mick-P-UK. Repository name: Dex-MickP-Personal
3. Visibility: PRIVATE (important - it will hold personal notes).
4. Do NOT add a README, .gitignore, or licence (we create locally).
5. Create repository. Copy the HTTPS URL:
   https://github.com/Mick-P-UK/Dex-MickP-Personal.git

### B2. Create the local backup repo (WHERE: PowerShell on your PC)

    $backup = "C:\Vaults\Dex-MickP-Personal"
    New-Item -ItemType Directory -Force -Path $backup | Out-Null
    Set-Location $backup
    git init
    git branch -M main
    git remote add origin "https://github.com/Mick-P-UK/Dex-MickP-Personal.git"

### B3. Add a safety .gitignore to the backup repo (WHERE: PowerShell on your PC)

This is a SECOND line of defence so secrets can never be pushed even if the mirror
list is wrong.

    $backup = "C:\Vaults\Dex-MickP-Personal"
    @"
    # Never back up secrets to GitHub
    .env
    .env.*
    **/.credentials/
    *.token
    *.pickle
    *google*credential*
    *google*token*
    "@ | Set-Content -Encoding ascii (Join-Path $backup ".gitignore")

### B4. Create the mirror-and-push script (WHERE: PowerShell on your PC)

Save this as C:\Vaults\backup-personal.ps1 . It mirrors the personal folders from the
live vault into the backup repo, then commits and pushes. It NEVER copies secrets.

    $vault  = "C:\Vaults\Mick's-Dex-2nd-Brain\Dex-MickP"
    $backup = "C:\Vaults\Dex-MickP-Personal"

    # Personal folders to back up (the gitignored PARA set + wrap output).
    $folders = @(
      "00-Inbox","01-Quarter_Goals","02-Week_Priorities","03-Tasks",
      "04-Projects","05-Areas","07-Archives","99-Private",
      "08-People","09-Entities","Active",
      "_handovers","_changelog","_daily"
    )

    foreach ($f in $folders) {
      $src = Join-Path $vault  $f
      $dst = Join-Path $backup $f
      if (Test-Path $src) {
        # /MIR mirror; exclude any credentials/secrets defensively
        robocopy $src $dst /MIR /XD ".credentials" /XF ".env" "*.token" "*.pickle" | Out-Null
      }
    }

    # Also back up the memory/session files if you want them here too (optional):
    Copy-Item (Join-Path $vault "CEDRIC_MEMORY.md") $backup -ErrorAction SilentlyContinue

    Set-Location $backup
    git add -A
    $stamp = Get-Date -Format "yyyy-MM-dd HH:mm"
    git commit -m "Personal backup: $stamp" 2>$null
    git push origin main

First manual run (WHERE: PowerShell on your PC):

    powershell -ExecutionPolicy Bypass -File "C:\Vaults\backup-personal.ps1"

You will be prompted for GitHub credentials on the first push (use a Personal Access
Token as the password, or GitHub CLI / credential manager).

### B5. Schedule it at 9pm (WHERE: Task Scheduler on your PC)

Match your existing Dex-MickP job:

1. Open Task Scheduler > Create Task.
2. Name: Backup Personal Content to GitHub.
3. Triggers: New > Daily > 21:00.
4. Actions: New > Start a program.
   Program: powershell.exe
   Arguments: -ExecutionPolicy Bypass -File "C:\Vaults\backup-personal.ps1"
5. Settings tab: TICK "Run task as soon as possible after a scheduled start is
   missed" (this is the same catch-up behaviour your current job uses).
6. Save.

  is still in git, so nothing is truly lost.
