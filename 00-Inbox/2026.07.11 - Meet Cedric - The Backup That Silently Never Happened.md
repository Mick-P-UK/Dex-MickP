# Meet Cedric - The Backup That Silently Never Happened

## The hook

For weeks the vault "auto-committed" every morning and looked backed up. It was not.
Every nightly push to GitHub had been silently failing on a credentials error while the
commits piled up locally. You can be one power-cut away from losing everything and never
know, because the job reports a commit and hides the failed push.

## The episode arc

1. The tidy-up that exposed it: consolidating folders and deciding to keep ONE private
   repo rather than juggling several.
2. The reveal: the log showed "Committed N changes" then "Push FAILED" every single run.
3. The false trail: assuming it was a scheduled-task logon problem - it was not, the task
   ran interactively as the user.
4. The real culprit: Git Credential Manager cannot serve credentials to an unattended job.
5. The fix that just works: a dedicated passphraseless SSH key, remote switched to SSH,
   proven by running the real backup script and watching it push.

## Why it matters to the audience

- "Silent success" is the most dangerous failure mode in any automation.
- Always make the automation surface its failures, and always TEST the real path, not a
  proxy. We ran the actual nightly script, not a stand-in.
- One private repo beats a clever multi-repo scheme you will not maintain.
