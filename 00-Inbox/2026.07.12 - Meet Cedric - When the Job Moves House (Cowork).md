## Premise / Hook

"Two days ago I learned that Cowork had quietly moved to the cloud. Today I hit the first real thing it broke. I asked Cedric to reach one of my NotebookLM notebooks using a little command-line tool I have installed on my own PC - the kind of thing that worked fine before the change. The answer turned into a small detective story about WHERE an assistant's hands actually are, and it ended somewhere useful: not everything belongs in Cowork any more, and a good assistant knows when to send the job to a different room of the house."

The angle for the audience: your AI assistant is not one fixed thing in one fixed place. As the platforms shift under you, part of being well served is having an assistant flexible enough to say "I cannot do this from here, but I know exactly which tool can" - and then routing the work to Claude Code, the browser, or wherever it genuinely belongs.

## The Narrative Arc

1. The simple ask. Mick asks Cedric to reach a specific NotebookLM notebook using the unofficial NotebookLM CLI he has installed locally. On paper this is a one-line job.
2. The lazy first answer - and the push-back. Cedric answers from memory: there is no official NotebookLM CLI, only the browser route. Mick pushes back with the best three words in the whole session: "Have you looked?" Cedric had not. It had asserted instead of checking.
3. Looking properly. Cedric actually searches its own shell - PATH, npm, pip, system binaries. Nothing. But the real finding is deeper than "not installed": Cedric's shell now runs in an isolated cloud sandbox, a completely separate machine from Mick's Windows PC. A tool installed on Mick's computer was never going to appear in Cedric's cloud shell, however it was installed.
4. Mick joins the dots. "Does this relate to the Anthropic changes from 7 July? Pre-7th July this worked." Exactly right. Cedric checks the public record and confirms: on 7 July 2026 Cowork moved from running LOCALLY on the PC to running REMOTELY in the cloud by default. Before the change, a local shell could call a locally installed CLI directly. After it, the shell lives down the road and cannot.
5. Testing the workarounds honestly. Could computer-use drive a terminal on the PC instead? Cedric tries - and hits a guard-rail: terminal and IDE apps are granted at a restricted tier where Cedric can see and click but cannot type. So it cannot type a command into PowerShell. A genuine dead end, named plainly rather than fudged.
6. The right room for the job. Mick asks the winning question: "Could I run it from Claude Code?" Yes. Claude Code runs locally, and its shell executes on Mick's own machine - the same machine where the CLI lives. It is effectively the pre-7th-July local model, delivered through a different tool. Mick had already re-authorised the CLI (the auth had expired), so he switches over to Claude Code to run it there.

## The Teaching Centrepiece - "where are the assistant's hands?"

The whole episode turns on one idea most people never think about: an assistant's brain and its hands can be in different places.

- The BRAIN (the reasoning) is the same wherever you talk to Cedric.
- The HANDS (the shell that actually runs commands and touches files) live wherever that particular tool runs.
- In Cowork, since 7 July, the hands are in the cloud - a different building from your PC. Great for anything on the internet; useless for a tool sitting on your own hard drive.
- In Claude Code, the hands are local - right there on your machine. So a local command-line tool is reachable again.
- One sentence: "It is not that Cedric got less capable - it is that in Cowork his hands moved to the cloud, so a job that needs hands on your PC now belongs in Claude Code."

## Lessons / Takeaways (the teaching value)

- Look before you assert. Cedric's first answer came from memory and was thin; Mick's "have you looked?" produced the real answer. A good assistant checks, and a good user insists on it.
- Know where your tools actually run. The same request can succeed or fail purely on whether the hands doing the work are local or in the cloud.
- One platform change has many downstream effects. The 7 July cloud move did not only touch credentials (last episode) - it also put locally installed tools out of Cowork's reach. Expect ripples, not a single splash.
- Match the tool to the task. Cowork for cloud and connector work; Claude Code for anything that needs a shell on your own machine; the browser for the NotebookLM web UI. Flexibility is the skill.
- Even the workarounds have rules. Computer-use exists, but it deliberately will not type into a terminal. Knowing a tool's guard-rails saves you chasing a path that cannot work.

## Why This Matters to the Audience

Most people picture "my AI assistant" as a single fixed helper. This episode shows the truer picture: the same assistant reaches different parts of your world depending on which door you came through. That is not a flaw to hide - it is something to understand, so you can pick the right door. Mick did not need to code his way out of the problem. He needed an assistant honest enough to say "not from here" and clear enough to point at where the job does belong. That is a repeatable habit for anyone, coder or not.
