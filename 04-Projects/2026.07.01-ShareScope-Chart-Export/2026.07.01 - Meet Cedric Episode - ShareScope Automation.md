## The Narrative Arc (build in steps)
1. Memory as a starting point. Cedric searched past sessions and found the old
   ShareScope chart-download work (Dec 2025 Playwright script + Jan 2026 Chrome test),
   plus a fuller modular automation project already in the vault.
2. A design decision, argued both ways. Chrome automation (slow, token-heavy) vs a
   headless Playwright script (fast, hands-off). Mick chose headless; Cedric agreed
   and dropped its earlier preference. Native "Save chart as PNG" chosen over a screen
   grab for a cleaner image.
3. Build the small discrete skill: sharescope-get-chart. Reused the proven login,
   search and logout building blocks; only the chart-view + save-PNG steps were new.
4. Test, do not trust. Three live runs, each peeling off one real-world selector:
   - Two buttons named "Chart" -> target the enabled one by data-cmd="ViewChart".
   - The 12-month period control is labelled "1 year".
   - The save item is "Save chart as PNG (bitmap)...", not the guessed "(Scaled)".
   Then it worked end to end - a 116 KB chart PNG.
5. The architecture insight from Mick: do not open and close ShareScope for every
   task - that churn looks bot-like and is slower. Wrap everything in ONE login.
   Cedric built sharescope_session.py: one login, many tickers/tasks, one logout.
   Confirmed on Hardide (HDD): chart + six financial CSVs in a single 26s session.
6. The payoff: the chart folded into the branded DIY Investors report template
   (sharescope-report skill), proven with a real Hardide research brief.

## Lessons / Takeaways (the teaching value)
- "Test, Don't Trust" in action - claims about a UI are confirmed by running it.
- Build in small, reusable blocks - the new skill reused proven login/search/logout.
- One session, not many - respect the service and avoid a bot-like pattern.
- Security hygiene - we found a password sitting in plain text in a committed file
  and flagged it as an urgent fix (good honest moment for the audience).
